# 전략 작성 가이드

> `strategy.md` 파일 하나로 매매 전략을 제어한다. 코드 수정 없이 전략만 바꾸면 된다.

---

## 핵심 개념

```
strategy.md (자연어)  →  Claude가 읽고 해석  →  매매 결정
```

- 전략은 **자연어로 작성**한다. Python 코드가 아니다.
- Claude가 매 실행마다 이 파일을 프롬프트로 읽어 판단한다.
- 전략 변경 시 `strategy_history` 테이블에 자동 기록된다.

---

## 현재 전략 구조

### 3단계 전략 체계

| 전략 | 매수 임계 | 목표 수익 | 손절 | 1회 금액 |
|------|:---------:|:---------:|:----:|:--------:|
| 🛡️ 보수적 | 70점 | +15% | -5% (하이브리드) | 자산의 10% |
| ⚖️ 보통 | 55점 | +10% | -5% (하이브리드) | 자산의 15% |
| 🔥 공격적 | 45점 | +7% | -3% (하이브리드) | 자산의 20% |

활성 전략은 `strategy.md` 첫 줄의 마커로 표시:
```markdown
## 활성 전략: 🛡️ 보수적 (conservative)
```

## 점수제 매수 시스템

"모든 조건 충족" 방식 대신, 각 조건에 점수를 부여하고 합산한다.

### 점수 배분 (보수적 기준)

| 조건 | 배점 | 충족 기준 | 부분 충족 |
|------|:----:|----------|----------|
| FGI (공포탐욕) | 30점 | ≤ 30 | ≤ 15이면 +5점 보너스 |
| RSI(14) | 25점 | ≤ 30 | 31~35이면 15점 |
| SMA(20) 이탈 | 25점 | -5% 이하 | -2.5%~-5%이면 15점 |
| 뉴스 감성 | 20점 | 부정적이지 않을 것 | — |

**예시:** FGI 28(30점) + RSI 33(15점) + SMA -6%(25점) = **70점 → 매수**

### 점수 변경 방법

대화형 세션에서 요청:
```
"매수 임계점을 60점으로 낮춰줘"
"RSI 기준을 35로 완화해줘"
"FGI 배점을 40점으로 올려줘"
```

## 매도 조건

### 일반 매도 (하나만 충족해도 매도)

| 조건 | 보수적 |
|------|--------|
| 목표 수익 | +15% |
| FGI | ≥ 75 (극도의 탐욕) |
| RSI | ≥ 70 (과매수) |

### 하이브리드 손절

손절선 도달 시 **무조건 매도하지 않는다**. 매수 시그널을 확인:

```
손절선 (-5%) 도달
    │
    ├── 매수 조건 3개+ 충족 (바닥 시그널)
    │   └── 손절 유예 + DCA (기존 보유량 50%, 1회만)
    │
    └── 매수 조건 2개 이하
        └── 즉시 손절

강제 손절선 (-10%) → 무조건 매도 (어떤 상황이든)
```

## 보조 필터

### AI 복합 시그널 (-85 ~ +85)

6가지 실시간 데이터를 종합한 점수:

```
호가 불균형 (±15) + 체결 압력 (±20) + 고래 방향 (±15)
+ 다중 TF (±10) + 변동성 (±10) + 거래량 (±15)
```

| 시그널 | 매수 시 | 매도 시 |
|--------|---------|---------|
| ≥ 0 | 매수 허용 | 매도 유예 가능 |
| < 0 | 매수 보류 | 즉시 매도 |
| ≤ -20 | 손절 유예 취소 | — |

### 온체인 역발상

| 지표 | 극단값 | 해석 |
|------|--------|------|
| 펀딩레이트 > 0.01% | 롱 과열 | 매수 보류 |
| 펀딩레이트 < -0.01% | 숏 과열 | 매수 긍정 |
| 롱숏비율 > 2.0 | 롱 극단 | 매도 신호 |
| 롱숏비율 < 0.5 | 숏 극단 | 매수 신호 |

### ADX 시장 레짐

| ADX | 레짐 | 포지션 배수 |
|-----|------|:-----------:|
| ≥ 25 | 추세장 | × 1.2 |
| 20~25 | 전환기 | × 0.8 |
| < 20 | 횡보장 | × 0.7 |

### ATR 포지션 사이징

| ATR 대비 평균 | 금액 조정 |
|:---:|:---:|
| > 1.5배 | × 0.6 |
| > 1.2배 | × 0.8 |
| 정상 범위 | × 1.0 |
| < 0.7배 | × 1.3 |

### Confidence 기반 사이징

| confidence | 금액 배수 |
|:---:|:---:|
| ≥ 0.85 | × 1.0 |
| 0.70~0.84 | × 0.7 |
| 0.55~0.69 | × 0.5 |
| < 0.55 | × 0.3 |

**최종 금액** = 기본 금액 × confidence × ATR × 레짐

## 전략 전환

### 전환 방법

대화형 세션에서:
```
"보수적 전략으로 바꿔줘"
"공격적 전략으로 전환해줘"
```

### 전환 기준 요약

| 방향 | 주요 트리거 |
|------|------------|
| → 🛡️ 보수적 | 지정학 위기, 매크로 악화, 3연속 손절, 빅 이벤트 전 |
| → ⚖️ 보통 | 리스크 해소 초기, 횡보장, 이벤트 통과 후 |
| → 🔥 공격적 | 바닥 확인 후 반등, 골든크로스, 강한 펀더멘탈 호재 |

### 전환 금지

- 급락 중 보수적 → 공격적 (FOMO 방지)
- 매매 후 4시간 이내
- 데이터 부족 시

## 초단타 전략 (별도 운영)

스윙과 **자금 분리**. `short_term_trader.py`가 실시간 실행.

| 전략 | 트리거 | 설명 |
|------|--------|------|
| 뉴스 반응 | RSS 감성 급변 | 0.5초 만에 감성 분석 → 선제 매매 |
| 급변동 리바운드 | 5분 내 1.5%+ 변동 | 되돌림 포착 |
| 고래 추종 | 500만원+ × 3건 연속 | 대량 체결 방향 추종 |

## 전략 수정 예시

```
# 조건 변경
"RSI 기준을 25에서 35로 완화해줘"
"FGI 조건을 40 이하로 바꿔줘"

# 규모 변경
"1회 매매를 자산의 15%로 올려줘"
"일일 최대 매매를 5회로 늘려줘"

# 손절 변경
"손절선을 -3%로 줄여줘"
"강제 손절을 -8%로 바꿔줘"

# 전략 전환
"보통 전략으로 바꿔줘"
```
