- RL 환경이 **1시간봉**으로만 훈련 → 초단타(1초~5분)와 중장기(4~8시간)의 실제 매매 주기와 불일치
- 에이전트 결정은 4~8시간 간격이지만, RL은 1시간 단위로 행동 → 과매매 발생
- 초단타 봇(±0.5% 익/손절, 최대 3분 보유)은 RL과 완전 별개

## 멀티 타임프레임 훈련 (신규 추가)

현재 1시간봉만 사용 → **4시간봉 환경 추가**로 에이전트 실행 주기와 정렬

### 왜 4시간봉인가?
- 에이전트 모드가 4~8시간 간격으로 실행
- 1시간봉 100K = 에피소드당 ~2,000스텝 → 과매매 유발 (269회 거래)
- 4시간봉 100K = 에피소드당 ~500스텝 → 거래 빈도 자연스럽게 하락
- 4시간봉은 노이즈가 적어 방향성 학습에 유리

### 훈련 매트릭스

| 타임프레임 | 데이터 | 스텝 | 용도 |
|-----------|--------|------|------|
| 1h (기존) | 180일 = 4,320봉 | 100K | 정밀한 진입/퇴출 타이밍 |
| 4h (신규) | 365일 = 2,190봉 | 100K | 트렌드 방향 판단 |

### 모델 앙상블 전략
- 1h 모델: action_1h (단기 타이밍)
- 4h 모델: action_4h (트렌드 방향)
- 최종 action = 0.4 * action_1h + 0.6 * action_4h
- 방향 불일치 시 관망 (action → 0)

## 보상 함수 v6 (확정)

```
reward = sharpe_reward + mdd_penalty + profit_bonus
```

| 구성요소 | 설명 | 범위 |
|---------|------|------|
| sharpe_reward | Differential Sharpe (윈도우 20) | [-1.5, +1.5] |
| mdd_penalty | 5% 초과 낙폭 × -2.0 | [-∞, 0] |
| profit_bonus | 수익성 거래 시 +0.1 | [0, 0.1] |
| risk_free_rate | 연 18% / 시간당 | 고목표 기준 |

## 연 18% 달성 전략

### 수학적 기초
- 연 18% = 월 1.39% (복리) ≈ 주 0.32% ≈ 일 0.046%
- 1시간봉 기준: 스텝당 0.002% 수익이면 달성
- MDD 5% 이내 유지 시 Sharpe ≈ 1.2 이상 필요

### RL 모델의 역할
- **직접 매매 X** → DecisionBlender에서 35% 가중치
- 에이전트(40%) + RL(35%) + LLM(25%) 융합
- RL은 "매수/매도 강도 제안"을 하고, 에이전트가 실행 여부를 최종 판단
- RL이 잘못된 방향을 제안해도 에이전트의 안전장치(danger_score)가 차단

### 리스크 관리
- MAX_POSITION_RATIO = 0.5 (최대 50% 투입)
- 자동 긴급정지: 4h -10% 급락, 연속손절 5회
- 주말 매매 축소 (외부지표 호조 시 무효화)
