## 기본 원칙 (모든 전략 공통)

- 대상: BTC/KRW (비트코인)
- 투자 성향: 보수적 (안전 우선)
- 1회 매매 금액: 최대 100만원 (매수/매도 동일)
- 일일 최대 매매 횟수: 3회
- 직전 매매 후 최소 4시간 경과해야 다음 매매 가능
- 급격한 가격 변동(1시간 내 5% 이상) 시에는 반드시 관망
- 뉴스에서 규제 관련 이슈 발견 시 매도 우선 고려
- 매매 실행 전 반드시 EMERGENCY_STOP, DRY_RUN, MAX_TRADE_AMOUNT 확인

## 전략별 매수/매도 조건 비교표

### 매수 조건 (점수제 — 임계점 이상 시 매수)

각 조건에 점수를 부여하고, 합산 점수가 임계점 이상이면 매수한다.
기존 "모두 충족" 방식은 서로 다른 타이밍에 충족되는 조건들이 동시에 만족하기 어려워, 점수 기반으로 전환한다.

| 조건 | 배점 | 🛡️ 보수적 기준 | ⚖️ 보통 기준 | 🔥 공격적 기준 |
|------|:----:|----------------|--------------|----------------|
| FGI | 30점 | ≤ 35 (공포) | ≤ 45 (공포) | ≤ 60 (중립 이하) |
| RSI(14) | 25점 | ≤ 35 (약세) | ≤ 40 (약세) | ≤ 50 (중립 이하) |
| SMA(20) 이탈 | 25점 | 3% 이상 하락 | 2% 이상 하락 | 1% 이상 하락 |
| 뉴스 감성 | 20점 | 부정적이지 않을 것 | 부정적이지 않을 것 | 무관 (자동 20점) |
| MACD | 보너스 | 무관 | 시그널선 근접/골든크로스 +10점 | 무관 |
| **외부 지표 (Data Fusion)** | **±20점** | strategy_bonus 값 직접 적용 | 동일 | 동일 |

| 전략 | 매수 임계점 | 설명 |
|------|:----------:|------|
| 🛡️ 보수적 | **60점** 이상 | 주요 2개 + 부분 충족 수준 (기존 70 → 60 하향: 놓친 기회 감소) |
| ⚖️ 보통 | **50점** 이상 | 주요 2개 충족 수준 (기존 55 → 50 하향) |
| 🔥 공격적 | **40점** 이상 | 주요 1개 + 부분 충족 수준 (기존 45 → 40 하향) |

> **v2 전략 조정 (2026-03-11)**: 30일 백테스트 결과, 관망 후 24h +2.32% 상승한 놓친 기회 10건 분석. FGI 극도공포(8~15)에서 보수적 임계점 70점이 너무 높아 매수 기회를 놓침. FGI/RSI 기준 완화 + 임계점 하향으로 대응.

**점수 부분 충족 규칙 (보너스):**
- FGI ≤ 기준의 경우: 기준의 50% 이하이면 추가 +5점 (예: 보수적에서 FGI ≤ 15이면 30+5 = 35점)
- RSI가 기준 근접 (기준+5 이내): 부분 점수 15점 (예: 보수적에서 RSI 33이면 25점 대신 15점)
- SMA 이탈이 기준의 절반 이상: 부분 점수 15점 (예: 보수적에서 SMA -3%이면 15점)

**외부 지표 Data Fusion 보너스 (±20점):**

`calculate_external_signal.py`가 산출하는 `strategy_bonus` 값을 매수 점수에 직접 가산한다.
4가지 외부 지표를 종합한 값이다:
- 바이낸스 파생상품 ±30 (롱숏 비율 + 펀딩비 + 김치 프리미엄)
- 거래소 내부 고래 ±15 (Upbit 1000만원+ 체결 방향)
- 시장 미시구조 ±15 (호가/체결/거래량)
- 온체인 고래 ±10 (블록체인 대형 이동 활동량)

| 종합 점수 | strategy_bonus | 의미 |
| +30 이상 | **+20점** | 외부 환경 극강세 (숏 과밀 + 디스카운트 + 고래 매수 집중) |
| +15 ~ +29 | **+10점** | 외부 환경 강세 |
| -14 ~ +14 | **0점** | 중립 |
| -29 ~ -15 | **-10점** | 외부 환경 약세 |
| -30 이하 | **-20점** | 외부 환경 극약세 (롱 과밀 + 프리미엄 + 고래 매도 집중) |

**사용자 피드백 (Bias) 적용 (중요):**
데이터 내 `사용자 수동 피드백 상태`의 `feedback_bias` 값이 존재할 경우, 이 값을 읽어서 최종 매수 점수에 직접 가산하거나 차감한다. (예: `feedback_bias`가 +0.5이면 매수 점수 +10점 가산, -0.8이면 매수 점수 -15점 차감, 즉 `bias * 20` 점수로 환산하여 적용). 이는 감독관의 개입을 가장 최우선으로 반영하기 위함이다.

| +20 ~ +29 | +15점 | 외부 환경 강세 |
| +10 ~ +19 | +10점 | 외부 환경 강세 |
| +5 ~ +9 | +5점 | 외부 환경 약한 강세 |
| -4 ~ +4 | 0점 | 중립 |
| -9 ~ -5 | -5점 | 외부 환경 약한 약세 |
| -19 ~ -10 | -10점 | 외부 환경 약세 |
| -29 ~ -20 | -15점 | 외부 환경 강한 약세 |
| -30 이하 | **-20점** | 외부 환경 극약세 (롱 과밀 + FOMO + 고래 매도 집중) |

**Data Fusion 겹침 규칙 (중요):**
- `fusion.signal`이 `strong_buy`이면: 4개 중 3개+ 지표가 동시 강세 → 매수 가중치 최대 강화
- `fusion.signal`이 `strong_sell`이면: 4개 중 3개+ 지표가 동시 약세 → 매도/관망 우선
- `fusion.signal`이 `mixed`이면: 강세/약세 혼재 → 관망 우선 고려
- 단일 지표만으로 매매 판단하지 않는다. 반드시 2개 이상 겹쳐야 가중치 부여

### 매도 조건 (하나라도 충족 시)

| 조건 | 🛡️ 보수적 | ⚖️ 보통 | 🔥 공격적 |
|------|-----------|---------|-----------|
| 목표 수익 | +15% | +10% | +7% |
| 손절선 | -5% (하이브리드) | -5% (하이브리드) | -3% (하이브리드) |
| 강제 손절 | -10% | -10% | -7% |
| FGI | ≥ 75 (극도의 탐욕) | ≥ 70 (탐욕) | ≥ 65 (탐욕 접근) |
| RSI(14) | ≥ 70 (과매수) | ≥ 65 (과매수 접근) | ≥ 60 (과매수 접근) |
| 볼린저 상단 이탈 | 무관 | 터치 시 경고 | 터치 시 매도 고려 |

### 포지션 과다 시 분할 매도 규칙 (모든 전략 공통)

BTC 포지션이 MAX_POSITION_RATIO(50%)를 초과한 경우, 목표 수익률을 낮춰 **보유량의 1/3을 분할 매도**하여 비중을 축소한다.

| 조건 | 기준 |
|------|------|
| 발동 조건 | BTC 비중 > 50% (MAX_POSITION_RATIO 초과) |
| 매도 트리거 | 평균매수가 대비 **+5%** 도달 시 |
| 매도 수량 | 보유량의 **1/3** |
| 목적 | 이익 확보 + 포지션 비중 축소 |
| 반복 | 매도 후에도 비중 50% 초과 시, 다음 +5% 구간에서 재발동 가능 |

**예시 (현재 상황):**
- 평단 103,340,261원 × 1.05 = **약 108,507,274원** 도달 시
- 0.0386 BTC × 1/3 = **약 0.0129 BTC 매도**
- 매도 후 BTC 비중 약 80% → 약 55%로 축소

**주의:** 포지션 과다 분할 매도는 일반 목표 수익 매도(+15%/+10%/+7%)보다 **우선 적용**된다. 비중 정상화가 리스크 관리의 최우선이다.

### 강화된 하이브리드 손절 규칙 (모든 전략 공통)

손절선 도달 시 **감독(Orchestrator)이 하락 컨텍스트를 종합 분석**하여 매도/물타기를 판단한다.

#### 감독의 하락 컨텍스트 5가지 분석 요소

| # | 요소 | 측정 방법 | 영향 |
|---|------|----------|------|
| 1 | **하락 속도** | 4h 가격변동 vs 24h 가격변동 비교 | 4h -3% 이상 = 급락 판정 |
| 2 | **하락 추세 지속** | 연속 음봉 수 + 12h 변동 + 거래량 동반 | 3봉+ 연속 음봉 = 추세 하락 |
| 3 | **DCA 이력** | state 파일에 DCA 횟수 추적 | 1회 실행 후 추가 물타기 금지 |
| 4 | **외부 약세 겹침** | 고래매도+펀딩+롱과밀+김치P+뉴스+매크로 | 3개+ 겹침 시 DCA 차단 |
| 5 | **캐스케이딩 위험** | 위 4가지 종합 점수 (0~100) | 70+ = 무조건 손절 |

#### 캐스케이딩 위험 점수 산출

| 요인 | 조건 | 점수 |
|------|------|:----:|
| 4h 급락 | < -3% | +30 |
| 4h 하락 | -1.5% ~ -3% | +15 |
| 하락 가속 | 4h 하락이 24h 하락의 50%+ | +20 |
| 거래량 급증+하락 | 거래량 2배+ & 하락 중 | +20 |
| 연속 음봉 | 5봉+ 연속 | +15 |
| 연속 음봉 | 3~4봉 연속 | +8 |
| 외부 약세 | 약세 지표 1개당 | +7 (최대 25) |

### 확신도 기반 포지션 사이징 (Confidence-Weighted)

매매 결정의 `confidence` 값에 비례하여 매매 금액을 조절한다:

| confidence 범위 | 금액 배수 | 의미 |
|:---:|:---:|------|
| ≥ 0.85 | × 1.0 | 강한 확신 → 기본 금액 전부 |
| 0.70 ~ 0.84 | × 0.7 | 보통 확신 → 70% |
| 0.55 ~ 0.69 | × 0.5 | 약한 확신 → 50% |
| < 0.55 | × 0.3 | 매우 불확실 → 30% (최소 진입) |

최종 매매 금액 = 기본 금액 × confidence 배수 × ATR 조정 × 레짐 조정

## 🛡️ 보수적 전략 (conservative) ← 현재 활성

> 폭락장 저점 매수만. 발동 빈도 낮지만 자산 보전 우선.

### 매수 조건 (점수제 — 70점 이상 시 매수)

| 조건 | 충족 시 점수 | 부분 충족 |
|------|:-----------:|----------|
| FGI ≤ 30 | 30점 | FGI ≤ 15이면 +5점 보너스 |
| RSI(14) ≤ 30 | 25점 | RSI 31~35이면 15점 |
| SMA(20) -5% 이하 | 25점 | SMA -2.5%~-5%이면 15점 |
| 뉴스 비부정 | 20점 | — |
| 외부 지표 Data Fusion | ±20점 | strategy_bonus 값 직접 적용 |

합산 70점 이상 → 매수 실행
(예: FGI 충족 30 + RSI 부분 15 + SMA 충족 25 = 70점 → 매수 가능)
(예: FGI 30 + RSI 15 + 뉴스 20 + 외부지표 +10 = 75점 → 매수 가능)

### 매도 조건 — 하이브리드 손절 전략

손절선 도달 시 무조건 매도하지 않고, **매수 시그널 상태를 확인하여 판단**한다.

#### 일반 매도 (하나라도 충족 시)
- 매수 대비 15% 이상 수익 (목표 수익)
- Fear & Greed Index ≥ 75 (극도의 탐욕)
- RSI(14) ≥ 70 (과매수)

#### 손절선 도달 시 (-5%) 분기 판단

**Case 1: 바닥 시그널 있음 → 손절 유예 + DCA**
- 조건: 매수 조건 3개 이상 충족 (FGI ≤30, RSI ≤30, SMA -5% 등)
- 판단 근거: 시스템이 "지금 사라"고 하는 구간에서 파는 것은 모순
- 행동:
  - 기존 보유분 유지 (손절 유예)
  - 추가 매수(DCA): 기존 보유량의 50% 이내, **1회만** 허용
  - 평균매수가를 낮춰 반등 시 수익 구간 확대
  - MAX_POSITION_RATIO(50%) 초과 금지

**Case 2: 바닥 시그널 없음 → 즉시 손절**
- 조건: 매수 조건 2개 이하 충족
- 판단 근거: 바닥이 아닌 하락 추세이므로 추가 손실 방지
- 행동: -5% 도달 즉시 시장가 매도

#### 강제 손절선 (-10%)
- Case 1(손절 유예) 상태라도 **-10% 도달 시 무조건 매도**
- 무한 홀드를 방지하는 절대 안전장치

## ADX 기반 시장 레짐 판단 (Market Regime)

ADX(Average Directional Index) 값으로 현재 시장이 추세장인지 횡보장인지 판단하고,
레짐에 따라 매매 전략의 가중치를 조정한다.

### 레짐 분류

| ADX 값 | 레짐 | 특성 |
|--------|------|------|
| ≥ 25 | **추세장 (Trending)** | 강한 방향성. MACD/EMA 크로스 신뢰도 높음 |
| 20~25 | **전환기 (Transitioning)** | 추세 시작/종료 가능. 조심스럽게 진입 |
| < 20 | **횡보장 (Ranging)** | 방향 없음. RSI/볼린저밴드 역추세 전략 유효 |

### 레짐별 전략 조정

#### 추세장 (ADX ≥ 25)
- **매수**: EMA(50) > EMA(200) (골든크로스 확인) + MACD 히스토그램 양수 → 추세 추종 매수
- **매도**: 추세 방향 역전 시그널 (MACD 데드크로스) 전까지 보유 연장
- **점수 보너스**: MACD 골든크로스 시 +10점 추가
- **포지션 크기**: 기본의 1.2배 (추세가 확인된 상태이므로)

#### 횡보장 (ADX < 20)
- **매수**: 볼린저 하단 터치 + RSI ≤ 30 → 반등 매수 (역추세)
- **매도**: 볼린저 상단 터치 + RSI ≥ 70 → 되돌림 매도
- **점수 보너스**: MACD 무시 (횡보장에서는 거짓 신호 빈번)
- **포지션 크기**: 기본의 0.7배 (방향성 불확실)

#### 전환기 (ADX 20~25)
- 기본 전략 유지, 보너스/감점 없음
- 포지션 크기: 기본의 0.8배

### ATR 기반 동적 포지션 사이징

ATR(Average True Range)을 사용하여 변동성에 따라 매매 금액을 자동 조절한다.

```
기본 매매 금액 = 총 자산의 N% (전략별 비율)
변동성 배수 = 기준 ATR / 현재 ATR
조정된 매매 금액 = 기본 매매 금액 × 변동성 배수
```

| ATR 대비 평균 | 조정 | 의미 |
|:---:|:---:|------|
| ATR > 평균 × 1.5 | 금액 × 0.6 | 고변동성 → 줄여서 리스크 관리 |
| ATR > 평균 × 1.2 | 금액 × 0.8 | 약간 높은 변동성 |
| 평균 범위 내 | 금액 × 1.0 | 기본 |
| ATR < 평균 × 0.7 | 금액 × 1.3 | 저변동성 → 늘려서 수익 극대화 |

**상한선**: 어떤 경우에도 `MAX_TRADE_AMOUNT`를 초과할 수 없다.

### 온체인 데이터 활용

선물 시장 온체인 데이터는 **역발상 지표**로 활용한다:

| 지표 | 극단값 | 해석 |
|------|--------|------|
| 펀딩레이트 > 0.01% | 롱 과열 | 단기 하락 조정 가능 → 매수 보류 |
| 펀딩레이트 < -0.01% | 숏 과열 | 숏스퀴즈 가능 → 매수 긍정 |
| 롱숏비율 > 2.0 | 롱 극단 | 역발상 매도 신호 |
| 롱숏비율 < 0.5 | 숏 극단 | 역발상 매수 신호 |
| OI 급증 + 가격 하락 | 숏 포지션 급증 | 반등 시 숏스퀴즈 가능 |
| OI 급감 + 가격 상승 | 포지션 청산 | 추세 약화 신호 |

## AI 복합 시그널 (AI Composite Signal)

사람이 동시에 처리할 수 없는 6가지 실시간 데이터를 AI가 종합하여 -100 ~ +100 점수로 산출한다.
스크립트: `scripts/collect_ai_signal.py`

### 구성 요소

| # | 분석 항목 | 점수 범위 | 측정 대상 |
|---|----------|:---------:|----------|
| 1 | 호가창 불균형 (Orderbook Imbalance) | +/-15 | 매수/매도 호가 총량 비율, 매수벽/매도벽 감지 |
| 2 | 체결 강도 (Trade Pressure) | +/-20 | 최근 200건 체결의 매수/매도 비율 |
| 3 | 고래 감지 (Whale Detection) | +/-15 | 100만원+ 대량 체결의 매수/매도 방향 |
| 4 | 다이버전스 (Multi-TF Divergence) | +/-10 | 일봉/4시간/1시간 RSI 괴리 |
| 5 | 변동성 레짐 (Volatility Regime) | +/-10 | 3일/20일 변동폭 비율 |
| 6 | 거래량 이상 (Volume Anomaly) | +/-15 | 거래량 급증 + 가격 방향 |

### 점수 해석

| 점수 | 해석 | 코드 |
|------|------|------|
| +30 이상 | 강한 매수 시그널 | `strong_buy` |
| +10 ~ +29 | 약한 매수 시그널 | `weak_buy` |
| -9 ~ +9 | 중립 | `neutral` |
| -29 ~ -10 | 약한 매도 시그널 | `weak_sell` |
| -30 이하 | 강한 매도 시그널 | `strong_sell` |

### 매매 결정에 적용하는 방법

AI 복합 시그널은 기존 매수/매도 조건의 **보조 필터**로 작동한다.

#### 매수 시 보조 필터
- 기존 매수 조건 모두 충족 + AI 복합 시그널 ≥ 0 → **매수 실행**
- 기존 매수 조건 모두 충족 + AI 복합 시그널 < 0 → **매수 보류** (더 좋은 타이밍 대기)
- 예외: AI 복합 시그널이 -10 ~ -1 범위이고 FGI ≤ 15(극단적 공포)이면 매수 허용

#### 매도 시 보조 필터
- 일반 매도 조건 충족 + AI 복합 시그널 ≤ 0 → **즉시 매도**
- 일반 매도 조건 충족 + AI 복합 시그널 > 20 → **매도 유예** (추가 상승 가능, 단 1회만)
- 강제 손절선 도달 시에는 AI 시그널과 무관하게 **무조건 매도**

#### 하이브리드 손절 시 보조 판단
- 손절선 도달 + 매수 조건 3개+ 충족 + AI 시그널 ≥ 0 → **손절 유예 + DCA 강력 권장**
- 손절선 도달 + 매수 조건 3개+ 충족 + AI 시그널 < -20 → **즉시 손절** (바닥 시그널이지만 실시간 매도 압력이 너무 강함)

## 전략 전환 — 감독 에이전트 자율 운영

에이전트 모드에서는 Orchestrator(감독)가 시장 상황을 **자율적으로** 판단하여 전략을 전환한다.
사용자 승인 없이 자동 전환되며, 모든 전환은 `agent_switches` 테이블에 기록되어 학습에 활용된다.

### 전환 판단 원리: 위험도/기회 이중 점수

감독은 매 실행마다 두 점수를 산출한다:
- **위험도(danger_score)**: 연속 손절, BTC 과다 보유, 급락, 김치P 과열, 롱 과밀 등을 가중 합산
- **기회(opportunity_score)**: 극공포, RSI 과매도, 반등, Data Fusion 강세, 숏 과밀 등을 가중 합산

두 점수의 조합으로 전략을 결정한다. 보수적↔공격적 직행도 허용한다.

### 대화형 수동 전환

대화형 세션에서도 다음과 같이 수동 전환 가능:
- "보수적 전략으로 바꿔줘"
- "공격적 전략으로 전환해줘"
- "보통 전략으로 돌아가줘"

---

### 🛡️ → 보수적으로 전환해야 하는 상황

#### 1. 지정학 리스크 (현재 해당)
- 전쟁 발발/확전, 군사 충돌, 경제 제재 확대
- 유가 급등 ($10+ 단기 상승), 호르무즈/수에즈 봉쇄 위협
- 핵심 교역 루트 차단, 글로벌 공급망 위협
- **현재 상황**: 이란-미국 충돌, 걸프만 유조선 공격, 유가 $80 급등

#### 2. 매크로 경제 위기
- 중앙은행 긴급 금리 인상, 인플레 급등
- 주요 주가지수 급락 (다우/나스닥 -3% 이상 단일일)
- 국채 수익률 급등 (10년물 50bp+ 단기 상승)
- 달러 인덱스 급등 (위험자산 회피)

#### 3. 암호화폐 고유 리스크
- 대형 거래소 파산/해킹 (FTX, Mt.Gox급)
- 주요국 암호화폐 전면 규제/금지 법안
- 스테이블코인 디페그 (USDT/USDC)
- 대규모 해킹으로 시장 패닉

#### 4. 포트폴리오 상황
- 총 자산 대비 BTC 비중 50% 초과
- 최근 3회 연속 손절 발생
- 계좌 잔고가 최초 투자금 대비 -20% 이하

#### 5. 빅 이벤트 직전
- FOMC 금리 결정 전날
- 미국 고용지표/CPI 발표 당일
- BTC 반감기 당일 (변동성 극대)
- 규제 관련 법원 판결/의회 표결 당일

---

### ⚖️ → 보통으로 전환해야 하는 상황

#### 1. 리스크 해소 초기
- 전쟁/분쟁 휴전 협상 시작
- 제재 완화 신호, 유가 안정화
- 규제 불확실성 해소 (법안 부결/완화)
- **전환 시점**: 악재가 "진행 중"에서 "해소 시작"으로 전환될 때

#### 2. 횡보장/방향 불분명
- FGI 30~60 중립 구간 지속 (1주일+)
- RSI 40~60 중립 구간
- BTC 가격이 SMA20 ±3% 내에서 등락
- 거래량 감소, 뚜렷한 추세 없음

#### 3. 완만한 조정
- 고점 대비 10~20% 하락 후 안정화
- 급락 없이 천천히 하락하는 패턴
- 매도 압력 감소, 호가 스프레드 축소

#### 4. 이벤트 통과 후
- FOMC/CPI 등 빅 이벤트 무난히 통과
- 예상 범위 내 결과로 시장 안도
- 변동성 축소, 정상 거래 패턴 복귀

#### 5. 포트폴리오 정상화
- 최근 매매 승률 50% 이상 회복
- 총 자산 안정화, 큰 손실 없음
- BTC 비중 20~40% 적정 수준

### 🔥 → 공격적으로 전환해야 하는 상황

#### 1. 바닥 확인 후 반등
- FGI 극도의 공포(≤20) → 3일 연속 상승으로 반등 확인
- 거래량 급증과 함께 가격 반등 (거래량 2배+ 증가)
- RSI 30 이하 → 40 돌파 (과매도 탈출)
- **핵심**: 단순 공포가 아니라 "바닥 확인"이 필요

#### 2. 기술적 강세 시그널
- 골든크로스: SMA(50)이 SMA(200) 상향 돌파
- 주요 저항선 돌파 후 지지선 전환 확인
- 상승 삼각형/역헤드앤숄더 패턴 완성
- MACD 골든크로스 + 히스토그램 양수 전환

#### 3. 강력한 펀더멘탈 호재
- BTC ETF 신규 승인 또는 대규모 자금 유입
- 반감기 3~6개월 전 (역사적 강세 구간)
- 대형 기관(테슬라, 마이크로스트래티지급) BTC 대량 매수 발표
- 주요국 암호화폐 우호적 법안 통과

#### 4. 시장 구조적 강세
- 주요 지수(S&P500, 나스닥) 신고가 경신
- 달러 약세 전환 (위험자산 선호)
- 금리 인하 시작 또는 확실한 인하 시그널
- 글로벌 유동성 확대 정책

#### 5. 공포 속 역발상 기회 (고급)
- FGI 극도의 공포인데 온체인 지표는 건전
  - 장기 보유자(LTH) 매도 없음
  - 거래소 BTC 보유량 감소 (인출 증가)
  - 채굴자 항복 후 해시레이트 회복
- 레버리지 청산 완료 후 시장 안정화
- **주의**: 이 전략은 경험이 필요하며, 잘못 판단하면 큰 손실 가능

## 권한 체계

```
👑 사용자 (최고 권한)
  ├─ .env EMERGENCY_STOP=true (수동 발동/해제)
  ├─ 웹 대시보드 토글
  └─ 대화형 세션 명령
      ※ 사용자가 건 긴급정지는 감독이 해제 불가

🎯 감독 Orchestrator (자율 권한)
  ├─ 전략 전환 (보수적/보통/공격적)
  ├─ 에이전트 결정 오버라이드 (DCA→매도, 매수→관망)
  ├─ 자동 긴급정지 발동 (조건 충족 시)
  │   └─ 발동 시: 전량 매도 + 매수 차단 + 텔레그램 알림
  └─ 자동 긴급정지 해제 (12시간 후 시장 안정 확인 시)
      ※ 사용자 수동 STOP과 독립 운영

🛡️⚖️🔥 에이전트 (실행 권한)
  ├─ 매수/매도/관망/DCA 결정
  └─ 감독의 임계값과 drop_context 내에서만 행동

🔒 execute_trade.py (최종 안전장치)
  ├─ 사용자 EMERGENCY_STOP → 모든 매매 차단
  ├─ 감독 자동긴급정지 → 매수만 차단 (매도/청산은 허용)
  └─ DRY_RUN, MAX_TRADE_AMOUNT 확인
```

### 감독 자동 긴급정지 발동 조건 (하나라도 충족 시)

| # | 조건 | 설명 |
|---|------|------|
| ① | 4h -10% 이상 | 플래시 크래시 |
| ② | cascade_risk ≥ 90 + danger ≥ 80 | 캐스케이딩 + 위험도 동시 극단 |
| ③ | 외부 약세 5개+ 동시 겹침 | 고래매도+펀딩+롱과밀+김치P+뉴스+매크로 전면 약세 |
| ④ | 연속 손절 5회+ AND 손실 -15%+ | 포트폴리오 위기 |

### 감독 자동 긴급정지 해제 조건 (모두 충족 시)

| 조건 | 기준 |
|------|------|
| 경과 시간 | 발동 후 최소 12시간 |

## AI 단타 트레이딩 (Short-Term Trading)

스윙 전략(8시간 간격)과 **별도로** 운영하는 실시간 단타 봇.
스크립트: `scripts/short_term_trader.py`

### 기본 원칙

- 스윙 전략 자금과 **완전 분리** (단타 전용 자금)
- 상시 실행 (tmux 윈도우에서 프로세스 유지)
- WebSocket으로 실시간 가격/체결 수신
- 수수료 왕복 0.1%를 반드시 초과하는 수익만 추구

### 안전장치

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| SHORT_TERM_BUDGET | 500,000원 | 단타 전용 자금 한도 |
| SHORT_TERM_MAX_TRADE | 200,000원 | 1회 최대 매매 금액 |
| SHORT_TERM_MAX_DAILY | 10회 | 일일 최대 매매 횟수 |
| SHORT_TERM_STOP_LOSS | 0.8% | 포지션별 손절선 |
| SHORT_TERM_TAKE_PROFIT | 0.8% | 포지션별 익절선 (수수료 차감 후 +0.7% 확보) |
| SHORT_TERM_MAX_HOLD_MIN | 20분 | 최대 보유 시간 (초과 시 자동 청산) |

### 3가지 전략

#### 1. 뉴스 반응 단타 (News Reaction)

Tavily로 3분마다 긴급 뉴스를 스캔하고, 감성 급변 시 선제 매매한다.

- **매수**: 강한 긍정 뉴스 (ETF 승인, 기관 매수 등) → 감성 점수 +0.4 이상
- **매도**: 강한 부정 뉴스 (해킹, 규제, 전쟁 등) → 감성 점수 -0.4 이하
- **AI 강점**: 뉴스 발생 후 0.5초 만에 감성 분석 완료 (사람은 30초~수분)

#### 2. 급등/급락 리바운드 단타 (Spike Rebound)

5분 내 1.5% 이상 급변동 후 되돌림을 포착한다.

- **급락 후 매수**: 1.5%+ 급락 → 0.3%+ 반등 확인 + 매수 체결 55%+ → 리바운드 매수
- **급등 후 매도**: 1.5%+ 급등 → 0.3%+ 되돌림 + 매도 체결 55%+ → 되돌림 매도
- **AI 강점**: 가격 변동 + 체결 방향 + 호가 변화를 동시에 분석하여 진짜 반등인지 판단

#### 3. 고래 추종 단타 (Whale Following)

500만원 이상 대량 체결이 3분 내 연속 3건 같은 방향이면 추종한다.

- **연속 매수 고래**: 3건+ 연속 대량 매수 → 따라 매수
- **연속 매도 고래**: 3건+ 연속 대량 매도 → 보유 시 청산
- **AI 강점**: 실시간 체결 스트림에서 대량 거래를 밀리초 단위로 감지

### 포지션 자동 관리

모든 포지션은 진입 즉시 자동 관리된다:

```
진입 → 매 1초마다 체크:
  +1.5% 도달 → 익절 (자동 매도)
  -0.8% 도달 → 손절 (자동 매도)
  30분 경과  → 시간 제한 청산 (현재 손익으로 청산)
```

### 실행 방법

```bash
# DRY_RUN 시뮬레이션

# 실매매 (DRY_RUN=false 필요)
```

### 스윙 전략과의 관계

| 항목 | 스윙 (기존) | 단타 (신규) |
|------|-----------|-----------|

| 보유 기간 | 시간~일 | 분 단위 (최대 20분) |

| 자금 | 메인 자금 | 별도 50만원 |
| 판단 기준 | FGI+RSI+SMA+뉴스 | 실시간 체결/호가/뉴스 |

두 전략은 자금이 분리되어 **서로 간섭하지 않는다.**
